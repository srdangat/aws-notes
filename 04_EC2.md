# AWS EC2 — Elastic Compute Cloud

## Overview

- **EC2 = Elastic Compute Cloud = Infrastructure as a Service**
- It mainly consists of the capability to:
  - Rent virtual machines **(EC2)**
  - Store data on virtual drives **(EBS)**
  - Distribute load across machines **(ELB)**
  - Scale the services using an auto-scaling group **(ASG)**

---

## EC2 User Data

- It is possible to **bootstrap** our instances using an **EC2 User Data** script
- **Bootstrapping** means launching commands when a machine starts
- That script is only run **once** at the instance's first start
- EC2 user data is used to **automate boot tasks** such as:
  - Installing updates
  - Installing software
  - Downloading common files from the internet
  - Anything you can think of
- The EC2 User Data Script runs with the **root user**

## EC2 Instance Types

### General Purpose
- Great for a **diversity of workloads** such as web servers or code repositories
- Balance between: **Compute, Memory, Networking**

### Compute Optimized
Great for **compute-intensive tasks** that require high performance processors:
- Batch processing workloads, Media transcoding
- High performance web servers
- High performance computing (HPC)
- Scientific modeling & machine learning
- Dedicated gaming servers

### Memory Optimized
Fast performance for workloads that **process large data sets in memory**:
- High performance relational/non-relational databases
- Distributed web scale cache stores
- In-memory databases optimized for BI (business intelligence)
- Applications performing real-time processing of big unstructured data

### Storage Optimized
Great for **storage-intensive tasks** that require high, sequential read and write access to large data sets on local storage:
- High frequency online transaction processing (OLTP) systems
- Relational & NoSQL databases
- Cache for in-memory databases (e.g., Redis)
- Data warehousing applications
- Distributed file systems

### Accelerated Computing
- GPU instances for ML, graphics, HPC

### Instance Naming Convention

Example: `m5.2xlarge`

| Part | Meaning |
|------|---------|
| `m` | Instance class |
| `5` | Generation (AWS improves them over time) |
| `2xlarge` | Size within the instance class |

## Security Groups

- Security Groups are the **fundamental of network security in AWS**
- They control how traffic is **allowed into or out of** our EC2 Instances
- Security groups only contain **ALLOW** rules
- Security group rules can **reference by IP or by security group**

### Key Properties
- Can be attached to **multiple instances**
- Locked down to a **region / VPC combination**
- Does live **"outside"** the EC2 — if traffic is blocked the EC2 instance won't see it
- It's good to maintain **one separate security group** for SSH access

### Default Behavior
- All **inbound** traffic is **blocked** by default
- All **outbound** traffic is **authorised** by default

### Common Ports

| Port | Protocol | Use |
|------|----------|-----|
| 22 | SSH | Log into a Linux instance |
| 21 | FTP | Upload files into a file share |
| 22 | SFTP | Upload files using SSH |
| 80 | HTTP | Access unsecured websites |
| 443 | HTTPS | Access secured websites |
| 3389 | RDP | Remote Desktop Protocol — log into a Windows instance |

### Troubleshooting
- If your application is **not accessible (time out)** → it's a **security group issue**
- If your application gives a **"connection refused"** error → it's an **application error** or it's not launched

## EC2 Pricing Models

| Pricing Model | Discount | Commitment | Best For | Key Feature |
|---------------|----------|------------|----------|-------------|
| **On-Demand Instances** | No discount | No commitment | Short-term or unpredictable workloads | Pay only for what you use |
| **Reserved Instances (RI)** | Up to 72% | 1 or 3 years | Stable workloads like databases | Reserve instance capacity |
| **Convertible Reserved Instances** | Up to 66% | 1 or 3 years | Workloads that may change | Can change instance type/family |
| **Savings Plans** | Up to 72% | 1 or 3 years | Flexible long-term usage | Commit to $/hour usage |
| **Spot Instances** | Up to 90% | No commitment | Fault-tolerant workloads | Uses unused AWS capacity |
| **Dedicated Instances** | Small discount possible | Optional | Workloads needing isolated hardware | Runs on dedicated hardware for one account |
| **Dedicated Hosts** | No major discount | Optional | Compliance & licensing requirements | Entire physical server dedicated to you |
